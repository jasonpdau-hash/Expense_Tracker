# Expense Tracker
An expense tracker application built off the FARM stack (fast api, react and mongodb). Users can add, remove and read their expense items. Administrators can monitor user actions including login and logout, as well as updates to their personal expense list.

## Application Features
- User authentication: User login utilises password hashing and JWT session management.
- Per-user expense tracking: Each user can access their own list of expenses.
- Admin dashboard: Admin accounts can access the event log to check user actions.
- Event logging: Login, logouts as well as add or delete expense items are recorded.
- Input validation: Inputs are validated at login and when inputing an expense item.
- Alerts and confirm dialogs: Feedback provided to user upon certain actions (such as logging out).
- Dynamic Webpage: Single page application that updates without needing to reload.

## Application stack
- Frontend: React
- Backend: FastAPI
- Database: MongoDB
- Hashing: Bcrypt
- Session: Json Web Token

## Database details
The application uses 3 mongo db collections.
- users - registered accounts and roles.
- expenses - record of expenses with title, amount, category, date, description and user's email.
- actions - a log of major actions in the application such as login, logout, and expense modifications.
